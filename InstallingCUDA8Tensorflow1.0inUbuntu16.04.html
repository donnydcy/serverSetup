Download two files from the folder: CUDA & cudnn/

cuda_8.0.61_375.26_linux.run

cudnn-8.0-linux-x64-v5.1.tgz

This procedure works fine on Geforce 650 Ti, which is also expected functional for later versions.

Pre-install cuda

Refer to http://www.pyimagesearch.com/2016/07/04/how-to-install-cuda-toolkit-and-cudnn-for-deep-learning/ with some edition:

  1. Disable the default  nouveau driver:

        $ sudo vim /etc/modprobe.d/blacklist-nouveau.conf

Add the following lines into the new file:

blacklist nouveau

blacklist lbm-nouveau

options nouveau modeset=0

alias nouveau off

alias lbm-nouveau off

Save this file, exit your editor, and then update the initial RAM filesystem, followed by rebooting your machine:

$ echo options nouveau modeset=0 | sudo tee -a /etc/modprobe.d/nouveau-kms.conf

$ sudo update-initramfs -u

$ sudo reboot

  1.  After rebooting, it may not be able to have a graphical desktop (gnome),  use Ctrl+Alt+F1 to switch into console mode and stop the x service:

        $ sudo service lightdm stop  or

        $ sudo service gdm stop        or

$ sudo service kdm stop

Install CUDA

  1. $ chmod +x cuda_8*.run
  2. $ mkdir installers
  3. $ sudo ./cuda_8*_linux.run -extract=`pwd`/installers  

Note the ` is the key left to “1”

  1. Change directory to  installers: $cd installers
  2.  Run following files in order:

$ sudo ./NVIDIA-Linux-x86_64*.run

$ modprobe nvidia

$ sudo ./cuda-linux64-rel-8*.run

$ sudo ./cuda-samples-linux-8*..run

  1. Add following lines in ~/.bashrc, if share for all users, add to /etc/bash.bashrc

# CUDA Toolkit

export CUDA_HOME=/usr/local/cuda-8

export LD_LIBRARY_PATH=${CUDA_HOME}/lib64:$LD_LIBRARY_PATH

export LD_LIBRARY_PATH=${CUDA_HOME}/extras/CUPTI/lib64:$LD_LIBRARY_PATH

export PATH=${CUDA_HOME}/bin:${PATH}

 

  1.  Update ~/.bashrc

$ source ~/.bashrc

Install cuDNN(Cudnn-8.0-linux-x64-v5.1.tgz)

Installing cuDNN is quite simple — all we need to do is copying the files in the lib64  and include  directories to their appropriate locations on the machine:

$ cd <cudnn path>
        $ tar -zxf cudnn-7.5-linux-x64-v5.0-ga.tgz

$ cd cuda

$ sudo cp lib64/* /usr/local/cuda/lib64/

$ sudo cp include/* /usr/local/cuda/include/

Install Tensorflow under Anaconda environment

https://github.com/tensorflow/tensorflow/blob/master/tensorflow/g3doc/get_started/os_setup.md#anaconda-installation

In Tensorflow-1.0 folder, there are different versions, in case its website crashing.

Google how to install .whl file using wheel.

  1. Create an environment called tf34 for python3.4 and tensorflow.

you can define the environment name (tf34) to anything you like.

Note that there are newer versions of Tensorflow which support Python3.5, 3.6, so the version of python is not necessary to be 3.4

$ conda create -n tf34 python=3.4 anaconda

  1. Activate the conda environment by issuing the following command:

        $ source activate tensorflow
         (tf34)$  # Your prompt should change

  1.  Save source url

(There is no link for gpu version in the official website, use the one which is provided in github):

(tf34)$ export TF_BINARY_URL=https://storage.googleapis.com/tensorflow/linux/gpu/tensorflow_gpu-1.0.0-cp34-cp34m-linux_x86_64.whl

  1.  Installation is straightforward:

        (tf34)$ pip install --ignore-installed --upgrade $TF_BINARY_URL

        Test:

        (tf34)$ python

        >>> import tensorflow as tf

       <there will be information which show successfully detecting CUDA devices>
        >>> hello = tf.constant('Hello, TensorFlow!')
        >>> sess = tf.Session()

       <Here will pop some warnings about CPU module, it’s fine>
>>> print(sess.run(hello))

  1.  After working in Tensorflow:

        (tf34)$ source deactivate
